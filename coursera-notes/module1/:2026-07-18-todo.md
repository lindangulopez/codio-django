# update when prject is done: 

The Django installation is fine. The failure is coming from your project settings:

```
File "/home/linda/GitLab_swh/blango/blango/settings.py", line 30
    X_FRAME_OPTIONS = 'ALLOW-FROM ' + os.environ.get('CODIO_HOSTNAME') + '-8000.codio.io'

TypeError: can only concatenate str (not "NoneType") to str
```

### What is happening

This line:

```python
os.environ.get('CODIO_HOSTNAME')
```

is returning `None` because the environment variable `CODIO_HOSTNAME` does not exist on your machine.

Python is effectively trying to do:

```python
'ALLOW-FROM ' + None + '-8000.codio.io'
```

which is invalid.

This project was likely created for **Codio** (a cloud IDE), where `CODIO_HOSTNAME` is automatically defined. You are running it locally, so that variable is missing.

---

## Fix option 1 (recommended): Make the setting local-friendly

Open:

```bash
nano blango/settings.py
```

Find:

```python
X_FRAME_OPTIONS = 'ALLOW-FROM ' + os.environ.get('CODIO_HOSTNAME') + '-8000.codio.io'
```

Replace it with:

```python
CODIO_HOSTNAME = os.environ.get('CODIO_HOSTNAME')

if CODIO_HOSTNAME:
    X_FRAME_OPTIONS = f'ALLOW-FROM {CODIO_HOSTNAME}-8000.codio.io'
else:
    X_FRAME_OPTIONS = 'DENY'
```

Save (`Ctrl+O`, Enter, `Ctrl+X`).

Then retry:

```bash
python manage.py migrate
```

---

## Fix option 2: Set the missing environment variable temporarily

If you want to mimic Codio:

```bash
export CODIO_HOSTNAME=localhost
```

Then:

```bash
python manage.py migrate
```

However, this is not ideal because Django's clickjacking protection setting is now pretending you are running inside Codio.
