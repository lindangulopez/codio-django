# Typical mistakes

## What this lesson is asking you to do

You are changing your Django project so the database settings can come from an **environment variable** instead of being hard-coded.

Right now your settings probably look like this:

```python
DATABASES = {
    'default': {
        'ENGINE': 'django.db.backends.sqlite3',
        'NAME': BASE_DIR / 'db.sqlite3',
    }
}
```

This works, but it is **locked to SQLite**.

The problem in production is that you may want:

* SQLite during development
* PostgreSQL on a server
* MySQL somewhere else

Instead of changing your code each time, you store the database information in an environment variable.

# Your mistake

Your error message:

```
ImportError: cannot import name 'TypedDict'
```

comes from this line:

```python
import dj_database_url
```

You installed:

```
dj-database-url-3.0.1
```

but your Codio environment uses:

```
Python 3.6
```

`dj-database-url 3.0.1` requires a newer Python version because it uses:

```python
TypedDict
```

which does not exist in Python 3.6.

So the problem is **not your Django code**.

The problem is:

```
Python 3.6
        +
new dj-database-url package
        =
incompatible
```

---

# Fix the package version

Because your course uses Django 3.2 and Python 3.6, install an older compatible version.

In the terminal:

```bash
pip3 uninstall dj-database-url
```

Then:

```bash
pip3 install dj-database-url==0.5.0
```

Check:

```bash
pip3 show dj-database-url
```

You should see:

```
Version: 0.5.0
```

# Your settings.py changes

At the top:

### Before:

```python
from configurations import Configuration, values
```

### Change to:

```python
from configurations import Configuration, values
import dj_database_url
```

---

## Replace your DATABASES section

You currently have:

```python
DATABASES = {
    'default': {
        'ENGINE': 'django.db.backends.sqlite3',
        'NAME': BASE_DIR / 'db.sqlite3',
    }
}
```

Replace it with:

```python
DATABASES = values.DatabaseURLValue(
    f"sqlite:///{BASE_DIR}/db.sqlite3"
)
```

---

## What this means

This line:

```python
DATABASES = values.DatabaseURLValue(
    f"sqlite:///{BASE_DIR}/db.sqlite3"
)
```

means:

"Use the database URL from the environment variable called `DATABASE_URL`. If it does not exist, use my SQLite database."

---

Example:

No environment variable:

```bash
python3 manage.py runserver 0.0.0.0:8000
```

uses:

```
sqlite:///db.sqlite3
```

---

Production example:

```bash
DATABASE_URL="postgres://user:password@server/mydb" python3 manage.py runserver
```

uses PostgreSQL instead.
